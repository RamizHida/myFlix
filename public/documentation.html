Documentation

myFlix API contains information about movies and directors. This information is stored using MongoDB. It includes basic CRUD functionality.

Business Logic Endpoint URL HTTP Method Request Response
Get a list off all movies /movies GET None A JSON object holding data on all movies. Example: [{ "movie-title" : "The Conguring" , "description" : "The Conjuring is...", "Genre" : {"name" "Horror" , "description" "Horror films are..."}, "Director" : {name: "James Wan" , "bio" : "....", "birthday": "1950-12-02": deathyear: "..."} , "imagePath" : "moviephotos.com/conguring" , "featured" : "true" },{...},...]
Get a list off all users /users GET None A JSON object holding data on all users. Example: [{"id" : "1asds92313" , "userName" : "John Doe", , "userEmail" : "john@jdoe.com", "favoriteMovies" : ["asdsad92323as", "sidnso123s"]}, {} ...]
Return data about a single movie by title /movies/[movie-title] GET None A JSON object holding data on a single movie. Example: { "movie-title" : "The Conguring" , "description" : "The Conjuring is...", "Genre" : {"name" "Horror" , "description" "Horror films are..."}, "Director" : {name: "James Wan" , "bio" : "James Wan is a ...", "birthday": "1950-12-02": deathyear: "..."} , "imagePath" : "moviephotos.com/conguring" , "featured" : "true" }
Return data about a genre by name /movies/genres/[genre-name] GET None A JSON object holding data on movie genre and description. Example: {"name": "Thriller", "description" : "Thriller movies are..."}
Return data about a director by name /movies/directors/[director-name] GET None A JSON object holding data on movie director. Example: {"name" : "James Wan" , "bio" : "James Wan is a ..." , "birthday" : "1950-12-02" , "deathyear" : "..."}
Allow new users to register /users POST A JSON object holding data about the user. Example : {"userName" : Sam Iam" , "password" : "password1234" , "userEmail" : "samiam@sam.com" , "userBirthDate" : "2000-10-17"} A JSON object holding data about the user that was added, including ID and empty favorite movie array. Example: {"userName" : Sam Iam" , "password" : "password1234" , "userEmail" : "samiam@sam.com" , "userBirthDate" : "2000-10-17", "_id" : "32e8sjd9s8d87", "favoiteMovies": []}
Allow users to update their user info /users/[user-name] PUT A JSON object holding the data you wish to be updated. Example: {"password": "fishylocks214"} A JSON Object holding the updated user info. Example: {"userName" : Sam Iam" , "password" : "fishylocks214" , "userEmail" : "samiam@sam.com" , "userBirthDate" : "2000-10-17", "_id" : "32e8sjd9s8d87", "favoriteMovies" : ["sd12...","lom12..."]}
Allow users to add a movie to their list of favorites /users/[user-name]/movies/[movie-id] POST None A JSON object holding user data, including updated favorite-movie list. Example: {"userName" : Sam Iam" , "password" : "fishylocks214" , "userEmail" : "samiam@sam.com" , "userBirthDate" : "2000-10-17", "_id" : "32e8sjd9s8d87", "favoriteMovies" : ["sd12...","lom12..."] }
Allow users to remove a movie from their list of favorites /users/[user-name]/movies/[movie-id] DELETE None A JSON object holding user data, including updated favorite-movie list. Example: {"userName" : Sam Iam" , "password" : "fishylocks214" , "userEmail" : "samiam@sam.com" , "userBirthDate" : "2000-10-17", "_id" : "32e8sjd9s8d87", "favoriteMovies" : ["sd12..."] }
Allow users to deregister /users/[user-name] DELETE None A text message indicating the account linked with the user has been deleted.
